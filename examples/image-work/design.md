画像作業 設計資料
=========================

目的
-------------------------

一般ユーザーが日常的に活用できるAIによる画像作業ユースケースを整理し、
最小限の入力で実用的な出力が得られる構成を定義する。

対象ユーザー
-------------------------

- 非エンジニア
- 一般的な業務従事者・個人事業主
- 日常的に画像を扱う個人ユーザー

設計の核心
-------------------------

writing と同じ思想を画像作業に適用する。

- ユーザーがやること: キーワード・テーマ・素材を投げるだけ
- AIがやること: ユースケースを文脈から判定し、登録済みファイルのルールに従って出力する
- ユーザーが毎回入力する必要があるもの: 原則として素材・キーワードのみ

プロンプトテンプレートで「スタイル・構図・色調・解像度…」を毎回入力させる設計は
このコンセプトと矛盾するため採用しない。

設計方針
-------------------------

### ユースケース別ファイル分割

画像作業の「良い出力」の基準はユースケースによって異なる。
イラスト生成と写真加工では判定基準・出力形式・AIへの指示の書き方がまったく違うため、
用途単位でファイルを分割し、それぞれに具体的なルールと出力例を持たせる。

### 全ファイル登録を前提とする

ユースケースが3つと少なく、`カスタム指示.md` と合わせて4ファイルで
ChatGPT 無料プランのファイル上限(5つ)に収まる。
writing のような棚卸し運用は不要で、全ファイルをまとめて登録する前提とする。
残り1枠はビジュアル設定ファイルの登録に充てることを推奨する。

### カスタム指示はAIへの動作命令として機能させる

「方針の説明文」ではなく「AIが従うべき命令」として記述する。
具体的には以下を定義する。

- ユースケースの自動判定ルール
- 不足情報があるときの推定・確認のルール
- 登録済みファイルの参照ルール

### ビジュアル設定ファイル(予備枠)

残り1枠に登録するファイル。以下の情報をまとめておくことで、
毎回の入力なしにAIが参照して処理を最適化する。

- 好みのテイスト・スタイル(例: フラットデザイン・アニメ風・写真風)
- イメージカラー・使用カラーコード
- SNSアカウント情報(例: Xアカウント → プロフィールアイコンの用途判定に使用)
- 用途・活動内容(例: 飲食店 → 料理写真の加工が多いと推定)

このファイルは `image-work` 単体で完結する内容として定義する。
`writing` のプロフィール・自社情報ファイルとは独立した別ファイルとして扱う。

ファイル構成
-------------------------

```
カスタム指示.md                  ← カスタム指示欄に貼り付ける。動作命令を記述する
README.md                        ← ユースケース概要・セットアップ手順・日常利用例
files/イラスト生成.md            ← スタイル判定・構図・色調・出力例
files/写真加工.md                ← 加工種別判定・強度・元素材の保持ルール・出力例
files/図解・ダイアグラム.md      ← 図の種類判定・構造・視認性ルール・出力例
```

予備枠1つにビジュアル設定ファイルを登録することを推奨する。
ビジュアル設定ファイルのテンプレートは README に記載する。

各ファイルに必須の記載事項
-------------------------

writing の実装を参照し、以下を必ず含める。

1. このファイルを参照するタイミング(AIへの指示)
2. 出力フォーマット定義
3. 判定ルール(スタイル・構図・加工種別などをAIが推定する基準)
4. 出力例(良い例・避けるべき例)
5. 出力手順(AIが従うステップ)

適用範囲と制約
-------------------------

- 対象: 日常的な画像作業(イラスト生成・写真加工・図解生成)
- 対象外: 高精度な商業印刷物・医療画像・法的効力を持つ図面
- 生成画像の著作権・利用規約はプラットフォームごとに異なるためユーザー責任で確認する
- 写真加工において元素材の著作権・肖像権の確認はユーザー責任とする
