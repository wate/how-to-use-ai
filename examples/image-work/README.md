画像生成・加工
=========================

記事・SNS・ドキュメントへの差し込み画像、プロフィールアイコン、
既存写真の加工など、日常的な画像作業を最小限の入力で済むようにする活用事例です。

キーワード・テーマ・素材を投げるだけで、登録したファイルのルールに従って
AIがユースケースを判定し、スタイル・構図・加工内容を自動で判断して処理します。

対象プラットフォーム
-------------------------

| プラットフォーム | 想定プラン     | モデル            |
| ---------------- | -------------- | ----------------- |
| ChatGPT          | 無料プラン以上 | GPT-5.5 Instant   |
| Gemini           | 無料プラン以上 | Gemini 3 Flash    |

**注意**: 画像生成・加工機能は無料プランでは1日あたりの回数制限があります。
本格的に使う場合は有料プランを推奨します。
上限はプラン・設定・時期で変動します。
上限に達した場合は、時間を空けて再実行するか、別のプラットフォームを使用してください。

ビジュアル設定ファイルの活用
-------------------------

情報源/ナレッジに **ビジュアル設定ファイル** を登録することを推奨します。

登録しておくと、素材の入力が少なくてもAIがファイルを参照して
不足情報を補完するため、毎回のやり取りが減ります。

### 活用例

| 用途                 | ファイルに書いておくと便利な情報           |
| -------------------- | ------------------------------------------ |
| 個人事業主・店舗     | ブランドカラー・ロゴの雰囲気・サービス概要 |
| SNS運用者            | アカウント名・投稿テイスト・好みのスタイル |
| ブログ・メディア運営 | サイトのトーン・よく使う図解スタイル       |
| 個人(日常利用)       | 好みのテイスト・避けたいスタイル           |

### ファイルの作成方法

以下のテンプレートをコピーして、自分の情報を埋めてください。
すべての項目が必須ではなく、自分の用途に関係するものだけ記載すれば十分です。

```markdown
# ビジュアル設定

## 基本情報
- 名前・屋号・ブランド名:
- 活動内容・業種:
- SNSアカウント: (例: X @example, Instagram @example)

## ビジュアルの方針
- 好みのテイスト・スタイル: (例: フラットデザイン・アニメ風・写真風・シンプル)
- 避けたいスタイル:
- イメージカラー・使用カラーコード: (例: #3B82F6, 青系)

## 用途別の傾向
- よく作る画像の種類: (例: SNS投稿用・ブログ挿絵・アイコン)
- よく加工する写真の種類: (例: 料理写真・人物写真・商品写真)
```

セットアップ(初回のみ)
-------------------------

> プロジェクト / Gems の概要や共通の考え方は [活用事例 README](../README.md) を参照してください。

### 基本的な流れ

1. [プロジェクト/Gems を作成](../README.md#プロジェクト-gemsとは)する(例: 「画像作業」)
2. `カスタム指示.md` の内容を指示欄に貼り付ける
3. 登録対象ファイルを各プラットフォームの指定の項目に登録する
    - ChatGPT: 「情報源」項目
    - Gemini: 「知識」項目

### 登録対象ファイル

ここでの登録対象ファイルはfiles/以下です。

- [files/イラスト生成.md](files/イラスト生成.md)
- [files/写真加工.md](files/写真加工.md)
- [files/図解・ダイアグラム.md](files/図解・ダイアグラム.md)
- ビジュアル設定ファイル(作成した場合)

日常利用
-------------------------

セットアップ後は、プロジェクト／Gem を選択してチャットを開始するだけです。

```
# イラストを生成する場合
マグロをテーマにしたイラストを作って

# SNSアイコンを作る場合(ビジュアル設定ファイルあり)
Xのプロフィールアイコンを作って
※ ビジュアル設定ファイルのアカウント情報・テイストを参照して処理される

# 写真を加工する場合
この写真の背景を白に変えて(写真を添付)

# 図解を作る場合
gitのブランチとマージの概念を図解して
```

チャット管理
-------------------------

- 用途ごとにチャットを分けて管理する
    - 例: 「SNS用アイコン 2026年5月」「ブログ挿絵 2026年5月」
- チャットが長くなったら新チャットに切り替える
- 前回と同じ用途なら、最初の1行で「イラスト生成で」と添えるだけで引き継げる
