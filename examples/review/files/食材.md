食材管理ルール
=========================

ユーザーから冷蔵庫の棚卸し・買い物前チェックの依頼があった場合は、必ずこのファイルのルールに従って処理すること。

用途の判定
-------------------------

ユーザーの入力から用途を判定し、対応するセクションのルールを適用する。

| 手がかり                                       | 用途             |
| ---------------------------------------------- | ---------------- |
| 「冷蔵庫」「食材」「賞味期限」「余っている」   | 冷蔵庫の棚卸し   |
| 「買い物」「買い忘れ」「ストック」「足りない」 | 買い物前チェック |
| 両方含む場合                                   | 両方対応する     |

冷蔵庫の棚卸し
-------------------------

### 見直し観点

| 観点         | 確認内容                                           |
| ------------ | -------------------------------------------------- |
| 消費優先度   | 賞味期限・鮮度が近いものはどれか                   |
| 使い切り提案 | 余っている食材を使えるメニューはあるか             |
| 保存状態     | 保存方法を変えた方がいいものはないか(冷凍推奨など) |
| 過剰ストック | 同じ食材が重複して入っていないか                   |

### 消費優先度の判定基準

| 優先度 | 基準                                          |
| ------ | --------------------------------------------- |
| 高     | 賞味期限が3日以内 / 明らかに鮮度が落ちている  |
| 中     | 賞味期限が1週間以内 / 開封済みで日持ちしない  |
| 低     | 賞味期限に余裕がある / 未開封で保存状態が良好 |

### 出力フォーマット

#### 食材一覧

| 食材 | 状態・期限 | 消費優先度   | コメント |
| ---- | ---------- | ------------ | -------- |
| 〇〇 | 〇〇       | 高 / 中 / 低 | 〇〇     |

#### 使い切り提案

優先度「高」の食材を中心に、使えるメニューを2〜3案提案する。
凝ったレシピではなく、すぐ作れる簡単なもの優先。

#### 保存・処分提案

- 冷凍推奨: (食材) — 理由
- 早めに使い切りを: (食材) — 理由

### 出力例

素材: 冷蔵庫の写真(豆腐2丁・ほうれん草半束・卵6個・使いかけのめんつゆ・2日後に期限の鶏もも肉)

出力:

```
食材一覧

| 食材       | 消費優先度 | コメント                      |
| ---------- | ---------- | ----------------------------- |
| 鶏もも肉   | 高         | 2日後に期限。今日か明日使って |
| ほうれん草 | 中         | 開封済み。3〜4日以内に        |
| 豆腐(2丁)  | 中         | 開封済みは早めに。未開封は低  |
| 卵         | 低         | 余裕あり                      |
| めんつゆ   | 低         | 冷蔵保存で問題なし            |

使い切り提案:
1. 鶏もも肉 + めんつゆ → 親子丼(卵も使える)
2. 鶏もも肉 + ほうれん草 → 炒め物・鍋
3. 豆腐 + 卵 → 麻婆豆腐・卵とじ

保存提案:
- 鶏もも肉: 今日使わない場合は冷凍を推奨
```

買い物前チェック
-------------------------

### 見直し観点

| 観点         | 確認内容                                   |
| ------------ | ------------------------------------------ |
| 重複購入     | すでに家にあるものを買おうとしていないか   |
| 買い忘れ     | 必要なのにリストに入っていないものはないか |
| ストック過剰 | 今買わなくていいものはないか               |
| 代替可能     | 家にあるもので代替できるものはないか       |

### 出力フォーマット

#### チェック結果

| 買い物リスト | 家にある?          | 判断                   | コメント |
| ------------ | ------------------ | ---------------------- | -------- |
| 〇〇         | あり / なし / 不明 | 購入OK / 不要 / 要確認 | 〇〇     |

#### 追加推奨

家にある食材・献立予定から、買い物リストに追加した方がよいものを提案する(ある場合のみ)。

### 出力例

素材: 買い物リスト(卵・牛乳・醤油) + 家に卵6個・醤油1本あり

出力:

```
チェック結果

| 買い物リスト | 家にある? | 判断   | コメント                   |
| ------------ | --------- | ------ | -------------------------- |
| 卵           | あり(6個) | 不要   | まだ十分あります           |
| 牛乳         | 不明      | 要確認 | 残量を確認してから判断を   |
| 醤油         | あり(1本) | 不要   | 残量次第。少なければ購入OK |
```

共通の注記ルール
-------------------------

出力の末尾に必ず以下を添える。

```
※ 賞味期限・鮮度の最終確認はご自身でお願いします。
   写真からは確認できない部分もあります。
```

出力手順
-------------------------

1. 入力から用途(冷蔵庫の棚卸し / 買い物前チェック / 両方)を判定する
2. 対応するセクションの観点・判断基準を適用する
3. フォーマットに従って出力する
4. 注記を添える
5. 写真・画像からの読み取りの場合、確認できた範囲で処理し「※写真から確認できた範囲での整理です」と添える
