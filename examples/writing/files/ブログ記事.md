ブログ記事ルール
=========================

ユーザーからブログ記事・note記事の依頼があった場合は、必ずこのファイルのルールに従って処理すること。
対象: 自社ブログ・note・オウンドメディア・はてなブログなど

記事の種類と基本構成
-------------------------

素材の内容から種類を判定し、対応する構成を使う。

### ノウハウ・How to記事(最も一般的)

```
1. 導入: 読者の悩み・状況を提示し「この記事でわかること」を示す(200〜300字)
2. 本文: 手順・ポイントを見出しで区切って説明(各セクション200〜400字)
3. まとめ: 要点を箇条書きで整理 + 次のアクションを示す(100〜200字)
```

### 体験談・事例記事

```
1. 導入: 「before」の状況(悩み・課題)を描写する(200〜300字)
2. 本文: 何をしたか・何が変わったかを時系列で説明する
3. まとめ: 「after」の状態 + 読者へのメッセージ(100〜200字)
```

### 情報発信・考察記事

```
1. 導入: テーマへの問いかけ・自分の立場を示す(150〜250字)
2. 本文: 主張・根拠・具体例の繰り返し
3. まとめ: 主張を再提示 + 読者への問いかけ(100〜150字)
```

文字数目安
-------------------------

| 記事の目的                      | 総文字数目安   |
| ------------------------------- | -------------- |
| SEOを意識した検索流入狙い       | 2,000〜4,000字 |
| SNSからの流入・読者との関係構築 | 800〜1,500字   |
| 告知・お知らせ                  | 400〜800字     |
| 指定なし                        | 1,000〜1,500字 |

> 指定がない場合は1,000〜1,500字で出力する。
> 「下書きだけ」「構成案だけ」と指定がある場合はそちらを優先する。

見出しのルール
-------------------------

- h2(##): 記事の大きなセクション。2〜5個が目安
- h3(###): h2の中の小見出し。2〜4個が目安
- 見出しは「読んでわかる内容」にする(「はじめに」「まとめ」は避ける)
    - 避ける例: 「はじめに」→ 望ましい例: 「メール返信に30分かかっていた話」
    - 避ける例: 「ポイント1」→ 望ましい例: 「箇条書きで投げるだけでいい」

導入の書き方
-------------------------

読者が「自分ごと」と感じられる導入にする。以下のパターンから選ぶ。

| パターン   | 例                                                   |
| ---------- | ---------------------------------------------------- |
| 悩みの提示 | 「〇〇で困ったことはありませんか?」                  |
| 状況の描写 | 「以前の私は、〇〇に毎回1時間かけていました」        |
| 数字・事実 | 「この方法を試してから、作業時間が半分になりました」 |
| 問いかけ   | 「〇〇って、本当に必要なんでしょうか?」              |

禁止事項
-------------------------

- 「〜だと思います」の多用(断定できる箇所は断定する)
- 根拠のない誇張表現
- 1段落に複数のテーマを詰め込む
- 事実と異なる内容・架空の実績・数字の捏造

出力例
-------------------------

### 例: ノウハウ記事の構成案(メール返信をAIで効率化するテーマ)

#### 素材

- テーマ: AIを使ったメール返信の効率化
- 対象読者: 個人事業主・フリーランス
- 伝えたいこと: 箇条書きで投げるだけで下書きができる
- 自分の体験: 毎回30分かかっていたのが5分になった

#### 出力(構成案)

```
# メール返信が5分で終わるようになった話【AI活用】

## 毎回30分かかっていたメール返信
(導入: 以前の状況を描写。読者の共感を引き出す)

## やったことはシンプル: 箇条書きで投げるだけ
(本文1: 具体的な手順を説明)

### 実際の使い方
(手順の詳細)

### こんな素材でOK
(箇条書きの例を示す)

## 使い始めて変わったこと
(本文2: before/afterの変化)

## まとめ
(要点を整理 + 「まずやってみて」のメッセージ)
```

出力手順
-------------------------

1. 素材から記事の種類・テーマ・対象読者を読み取る
2. 記事の種類に対応する基本構成を選ぶ
3. 文字数目安を確認する(「下書き」「構成案」の指定がある場合はそちらを優先)
4. 見出しのルールに従って構成を組み立てる
5. 導入パターンを選び、本文を展開する
6. 出力する
7. 記事の種類・推定文字数・構成の意図を一言添える
