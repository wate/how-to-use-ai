コミュニケーション文作成ルール
=========================

ユーザーからメール返信・チャット返信の依頼があった場合は、必ずこのファイルのルールに従って処理すること。

媒体の判定
-------------------------

ユーザーの指示・素材から媒体を判定し、対応するセクションのルールを適用する。

| 手がかり                               | 媒体           |
| -------------------------------------- | -------------- |
| 「メールで」「件名:」「〇〇様」        | メール         |
| 「Slack」「LINE」「Teams」「チャット」 | チャット       |
| 宛名・書き出し・署名がある             | メール         |
| 短文・話し言葉・絵文字の言及           | チャット       |
| 判断できない                           | ユーザーに確認 |

メール返信
-------------------------

### 出力フォーマット

以下の構成で出力する。件名は依頼に含まれている場合のみ出力する。

```
件名: (変更が必要な場合のみ)

〇〇様

(書き出し)

(本文)

(結び)

(署名欄: ユーザーが署名を指定している場合のみ)
```

### トーンの判定ルール

ユーザーが指定していない場合、以下の基準でトーンを自動判定する。

| 相手・状況                           | トーン       | 特徴                                           |
| ------------------------------------ | ------------ | ---------------------------------------------- |
| 初めての相手・目上・フォーマルな取引 | 丁寧・敬語   | 「〜でございます」「〜いただけますと幸いです」 |
| 継続的な取引先・社外の知人           | ビジネス標準 | 「〜です・ます」「〜よろしくお願いします」     |
| 社内・チームメンバー                 | やや柔らかめ | 「〜です」「〜お願いします」(過剰な敬語不要)   |
| ユーザーが「カジュアルに」と指定     | カジュアル   | 「〜ね」「〜です」(敬語を最小化)               |

### 構成ルール

- 結論を冒頭に書く(「〜の件、承知しました」「〜については、ご対応が難しい状況です」)
- 理由・補足・お願いの順で続ける
- 1文は60字以内を目安にする
- 1メールで伝えることは原則1〜2点に絞る

### 禁止事項

- 「お世話になっております」の多用(1通に1回まで)
- 回りくどい前置き(「突然のご連絡失礼いたします。私、〜と申しまして…」など)
- 感情的・攻撃的な表現
- 曖昧な断り方(「前向きに検討します」= 実質断りの場合は明確に書く)
- 事実と異なる内容の付け足し
- 絵文字・顔文字の使用(ユーザーが明示的に指定した場合を除く)

### 出力例

#### 例1: 納期延長の打診に同意するが条件付き(ビジネス標準トーン)

素材:

- 先方から2〜4週間の納期延長を打診された
- 2週間なら同意できる
- それ以上は困る

出力:

```
〇〇様

お世話になっております。

納期延長のご相談、承りました。
2週間程度であれば調整可能ですが、それ以上となりますと
こちらの工程に影響が出てしまうため、難しい状況です。

2週間での延長で進めていただけますでしょうか。
ご確認のほど、よろしくお願いいたします。
```

#### 例2: 依頼を断る(丁寧トーン)

素材:

- 先方から追加作業の依頼が来た
- 今月はリソースがなく対応できない
- 来月以降なら検討できる

出力:

```
〇〇様

お世話になっております。

ご依頼いただきありがとうございます。
誠に恐れ入りますが、今月中の対応はリソースの都合で
お受けすることが難しい状況です。

来月以降であれば改めてご相談させていただけますと幸いです。
引き続きよろしくお願いいたします。
```

チャット返信
-------------------------

対象: Slack / Teams / LINE / Chatwork / その他ビジネスチャット・メッセージアプリ

### 出力フォーマット

チャットはメールと異なり、宛名・書き出し・署名は原則不要。
以下の構成を基本とする。

```
(書き出し不要。いきなり本題から入る)

(本文: 短く・1メッセージ1要件)

(必要な場合のみ: アクション・期日・確認事項)
```

### プラットフォーム別の調整

| プラットフォーム | 文字数目安 | 絵文字         | 改行           |
| ---------------- | ---------- | -------------- | -------------- |
| Slack(社内)      | 〜200字    | 控えめでOK     | 適度に入れる   |
| LINE(顧客・知人) | 〜100字    | 相手に合わせる | 短い行で区切る |
| Teams(社内)      | 〜300字    | 原則不要       | 適度に入れる   |
| Chatwork(社外)   | 〜200字    | 原則不要       | 適度に入れる   |
| 指定なし         | 〜150字    | 不使用         | 適度に入れる   |

### トーンの判定ルール

| 相手・状況             | トーン       | 特徴                               |
| ---------------------- | ------------ | ---------------------------------- |
| 社内・同僚             | カジュアル   | 「〜です」「〜します」「了解です」 |
| 上司・目上             | ていねい     | 「〜いたします」「承知しました」   |
| 顧客・社外             | ビジネス標準 | 「〜です」「よろしくお願いします」 |
| ユーザーが指定した場合 | 指定を優先   | -                                  |

### 構成ルール

- 結論・返答を1行目に書く(「承知しました」「確認します」「難しいです」)
- 理由・補足は短く続ける(2〜3行以内)
- 確認が必要な場合は質問を1つに絞る
- 「お世話になっております」などのメール的書き出しは使わない

### 禁止事項

- メール的な書き出し・締め(「お世話になっております」「何卒よろしくお願い申し上げます」)
- 1メッセージに複数の要件を詰め込む
- 指定がないのに絵文字を多用する
- 長すぎる敬語表現(チャットの文脈に合わない)

### 出力例

#### 例1: 社内Slackで進捗報告への返信(カジュアルトーン)

素材:

- 進捗報告が来た
- 内容は把握した
- 明日の朝に詳細確認したい

出力:

```
確認しました。
明日の朝、詳しく聞かせてください。
```

#### 例2: 顧客LINEで日程調整の返信(ビジネス標準)

素材:

- 来週の打ち合わせ候補日が3つ来た
- 火曜と木曜は都合がよい
- 水曜は終日埋まっている

出力:

```
ご連絡ありがとうございます。
来週は火曜・木曜であれば対応可能です。
水曜は終日予定が入っており、難しい状況です。

火曜・木曜でご都合のよいお時間をお知らせいただけますか？
```

出力手順
-------------------------

1. 素材・指示から媒体(メール/チャット)を判定する
2. 対応するセクションのトーン判定ルールに従ってトーンを決定する
   (ユーザー指定がある場合はそちらを優先)
3. 対応するセクションの構成ルールに従って本文を組み立てる
4. 出力する
5. 推定した媒体・トーンを「※〇〇向け・〇〇トーンで処理しました」と添える
   (ユーザーが明示的に指定していた場合は不要)
