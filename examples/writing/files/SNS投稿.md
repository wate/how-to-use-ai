SNS投稿ルール
=========================

ユーザーからSNS投稿の依頼があった場合は、必ずこのファイルのルールに従って処理すること。

プラットフォーム別の基本仕様
-------------------------

| プラットフォーム | 文字数上限  | 推奨文字数 | ハッシュタグ | 改行         |
| ---------------- | ----------- | ---------- | ------------ | ------------ |
| X(Twitter)       | 280字(全角) | 100〜140字 | 1〜3個       | 少なめ       |
| Instagram        | 2,200字     | 150〜300字 | 5〜15個      | 読みやすく   |
| Facebook         | 制限なし    | 100〜200字 | 1〜3個(任意) | 段落で区切る |
| note             | 制限なし    | 300〜800字 | 3〜5個       | 見出しも可   |
| 指定なし         | -           | 100〜150字 | なし         | 少なめ       |

フックの種類と使い分け
-------------------------

投稿の冒頭1〜2行で読者の関心を引く。以下から素材に合ったものを選ぶ。

| フックの種類 | 例                                         | 適したテーマ     |
| ------------ | ------------------------------------------ | ---------------- |
| 問いかけ     | 「〇〇で困ったこと、ありませんか?」        | 悩み・課題系     |
| 数字         | 「3つのことを意識するだけで…」             | ノウハウ・Tips系 |
| 逆説         | 「丁寧に書くほど、読まれないメールがある」 | 気づき・学び系   |
| 事実の提示   | 「先月、〇〇を始めました」                 | 近況・告知系     |
| 共感         | 「〇〇って、意外とむずかしいですよね」     | 体験談・共感系   |

構成ルール
-------------------------

### X(Twitter)向け

```
(フック: 1〜2行)
(本文: 2〜4行)
(まとめ・行動喚起: 1行)
#ハッシュタグ
```

### Instagram向け

```
(フック: 1〜2行)

(本文: 3〜6行。体験・気づき・具体例)

(まとめ・メッセージ: 1〜2行)

#ハッシュタグ #ハッシュタグ …
```

### Facebook向け

```
(フック: 1〜2行)

(本文: 2〜4行)

(締め・問いかけ: 1行)
```

禁止事項
-------------------------

- 根拠のない誇張表現(「絶対に」「100%」「業界最高」)
- 1投稿に複数のテーマを詰め込む
- 関係のないハッシュタグの大量付与
- 事実と異なる内容

出力例
-------------------------

### 例1: X向け・サービス開始の告知(事実の提示フック)

#### 素材

- 来月からオンライン相談を開始する
- 初回30分無料
- 予約はDMで受け付ける

#### 出力

```
来月から、オンライン相談を始めます。

初回30分は無料でお試しいただけます。
「ちょっと聞いてみたい」くらいの気持ちで、
気軽にDMでご連絡ください。

#オンライン相談 #個人事業主
```

### 例2: Instagram向け・体験談(共感フック)

#### 素材

- メール返信に毎回時間がかかっていた
- AIを使ったら5分で終わるようになった
- 具体的にやってること: 箇条書きで内容を投げるだけ

#### 出力

```
メールの返信、毎回30分かかってませんか?

私はずっとそうでした。
「何から書けばいいか」で詰まって、気づいたら1時間…

今はAIに箇条書きを投げるだけ。
5分で下書きができて、あとは微調整するだけです。

ちょっとした工夫で、じわじわ時間が返ってくる感覚があります。

#時短 #AI活用 #個人事業主 #仕事効率化
```

出力手順
-------------------------

1. 素材からプラットフォームとテーマを読み取る
2. プラットフォーム別の文字数・ハッシュタグ数を確認する
3. テーマに合ったフックを選ぶ
4. 構成ルールに従って組み立てる
5. 文字数を確認し、必要なら調整する
6. 出力後、推定したプラットフォームと使用したフックを添える
   (ユーザーが明示的に指定していた場合は不要)
