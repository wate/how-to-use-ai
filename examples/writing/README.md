文章作成
=========================

メール返信・SNS投稿・PR文・ブログ記事など、日常的な文章作成を
最小限の入力で済むようにする活用事例です。

素材(箇条書き・メモ・下書き)を投げるだけで、登録したファイルのルールに従って
AIが構成・トーン・文字数を自動で判断して出力します。

対象プラットフォーム
-------------------------

| プラットフォーム | 想定プラン     | モデル          |
| ---------------- | -------------- | --------------- |
| ChatGPT          | 無料プラン以上 | GPT-5.3 Instant |
| Gemini           | 無料プラン以上 | Gemini 3 Flash  |

ファイル構成
-------------------------

| ファイル                      | 役割                                           |
| ----------------------------- | ---------------------------------------------- |
| `カスタム指示.md`             | カスタム指示欄に貼り付ける内容                 |
| `files/コミュニケーション.md` | メール返信・チャット返信の構成・トーン・出力例 |
| `files/発信・PR.md`           | SNS投稿・PR文の構成・フック・禁止表現・出力例  |
| `files/ブログ記事.md`         | ブログ記事の見出し構成・導入パターン・出力例   |
| `files/要約・変換.md`         | 要約・言い換え・トーン変換の各ルール・出力例   |

どのファイルを登録するか
-------------------------

ChatGPT 無料プランはプロジェクトへの登録ファイルが **5つまで** です。
全ファイルを登録する必要はありません。以下の棚卸し表を参考に、
自分が使うユースケースのファイルだけ選んでください。

**注意**: 後述する「プロフィール・自社情報ファイル」を登録する場合は、その分の枠も考慮して選んでください。

### ユースケース棚卸し表

| 自分の用途                                 | 登録するファイル        |
| ------------------------------------------ | ----------------------- |
| 取引先・顧客へのメール返信                 | `コミュニケーション.md` |
| 社内チャット・Slack・LINE での返信         | `コミュニケーション.md` |
| X(Twitter)・Instagram・Facebook への投稿   | `発信・PR.md`           |
| 商品・サービスの紹介文・ランディングページ | `発信・PR.md`           |
| 自社ブログ・note・オウンドメディアへの投稿 | `ブログ記事.md`         |
| 長い文章の要約・言い換え・トーン変換       | `要約・変換.md`         |

#### 例: 個人事業主(顧客対応＋ホームページ運営＋ブログ発信)の場合

```
登録するファイル:
- コミュニケーション.md  ← 顧客とのやり取りがメール中心
- 発信・PR.md            ← 商品・サービスページの紹介文
- ブログ記事.md          ← 活用事例や情報発信
- プロフィール情報.md    ← 自分で用意するプロフィール・自社情報ファイル(後述)

計4ファイル → 無料プランでも予備枠が1つ残る
```

プロフィール・自社情報ファイルの活用(予備枠の使い方)
-------------------------

4つのルールファイルに加えて、予備枠1つに **プロフィール・自社情報ファイル** を
登録することを推奨します。仕事・プライベートどちらの用途にも活用できます。

登録しておくと、素材の入力が少なくてもAIがファイルを参照して不足情報を補完するため、
ヒアリングのやり取りが減ります。

### 活用例

| 用途           | ファイルに書いておくと便利な情報                             |
| -------------- | ------------------------------------------------------------ |
| 個人事業主     | 屋号・サービス概要・ターゲット・よく使うハッシュタグ         |
| 会社員         | 部署・役職・よく連絡する相手の関係性・社内用語               |
| 個人(趣味発信) | ニックネーム・発信テーマ・好みのトーン・フォロワー層         |
| 個人(日常利用) | 自分の文体の好み・よく使う表現・避けたい表現                 |

### ファイルの作成方法

以下のテンプレートをコピーして、自分の情報を埋めてください。
すべての項目が必須ではなく、自分の用途に関係するものだけ記載すれば十分です。

```markdown
# プロフィール・自社情報

## 基本情報
- 名前・屋号:
- 活動内容・仕事内容:
- 活動エリア・拠点:
- ウェブサイト・SNSアカウント:

## ターゲット・相手
- メインターゲット・よく連絡する相手:
- 相手の主な悩み・関心:

## 商品・サービス・活動内容
### (商品・サービス・活動名)
- 概要:
- 特徴・強み:
- 価格帯(あれば):
- 実績・事例(あれば):

## トーン・表現方針
- 基本トーン: (例: 親しみやすく・丁寧に / プロフェッショナルに など)
- 使いたい表現:
- 避けたい表現・禁止ワード:

## その他
- よく使うハッシュタグ:
- 発信テーマ・キーワード:
```

### プロフィール・自社情報ファイルを登録した場合の動作

素材が「○○のSNS投稿を作って」だけであっても、
AIがファイルのターゲット・活動内容・トーンを参照して処理を開始します。
ファイルに記載がない情報が必要な場合のみ、確認を求めます。

セットアップ(初回のみ)
-------------------------

### ChatGPT(プロジェクト機能)

1. プロジェクトを作成する(例: 「文章作成」)
2. `カスタム指示.md` の内容をプロジェクトのカスタム指示欄に貼り付ける
3. 棚卸し表で選んだファイルと、プロフィール・自社情報ファイル(作成した場合)を情報源として登録する

### Gemini(Gems 機能)

1. Gem を作成する(例: 「文章作成」)
2. `カスタム指示.md` の内容を指示欄に貼り付ける
3. 棚卸し表で選んだファイルと、プロフィール・自社情報ファイル(作成した場合)をナレッジファイルとして登録する

日常利用
-------------------------

セットアップ後は、プロジェクト／Gem を選択してチャットを開始するだけです。

```
# メール返信を作る場合
以下の内容でメール返信を作って

- 先方からの納期延長の打診に同意する
- ただし2週間以上の延長は困る旨を伝えたい
- 丁寧だが率直なトーンで

# SNS投稿を作る場合(プロフィール・自社情報ファイルあり)
新商品のX向け告知投稿を作って
※ プロフィール・自社情報ファイルの情報を参照して処理される

# 既存の文章をリライトする場合
このメールをもっと簡潔にして(文章を貼り付け)
```

チャット管理
-------------------------

- 用途ごとにチャットを分けて管理する
    - 例: 「メール返信 2026年5月」「ブログ下書き 2026年5月」
- チャットが長くなったら新チャットに切り替える
- 前回と同じ用途なら、最初の1行で「メール返信で」と添えるだけで引き継げる
