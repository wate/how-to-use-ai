文章作成 設計資料
=========================

目的
-------------------------

一般ユーザーが日常的に活用できるAIによる文章作成ユースケースを整理し、
最小限の入力で実用的な出力が得られる構成を定義する。

対象ユーザー
-------------------------

- 非エンジニア
- 一般的な業務従事者・個人事業主
- 日常的に文章を扱う個人ユーザー

設計の核心
-------------------------

money-management と同じ思想を文章作成に適用する。

- ユーザーがやること: 素材(箇条書き・メモ・下書き)を投げるだけ
- AIがやること: ユースケースを文脈から判定し、登録済みファイルのルールに従って出力する
- ユーザーが毎回入力する必要があるもの: 原則として素材のみ

プロンプトテンプレートで「目的・ターゲット・トーン・制約…」を毎回入力させる設計は
このコンセプトと矛盾するため採用しない。

設計方針
-------------------------

### ユースケース別ファイル分割

文章作成の「良い出力」の基準はユースケースによって異なる。
メール返信とSNS投稿では文字数・構成・トーンの定義がまったく違うため、
一枚のファイルにまとめると判定基準が曖昧になる。
用途単位でファイルを分割し、それぞれに具体的なルールと出力例を持たせる。

### ファイルは「必要なものだけ登録」を前提とする

ChatGPT 無料プランのプロジェクトはファイル上限が5つ。
全ファイルを一括登録させる設計ではなく、
ユーザーが自分のユースケースを棚卸しして必要なファイルだけ選ぶ運用を前提とする。
README にその導線を設ける。

### カスタム指示はAIへの動作命令として機能させる

「方針の説明文」ではなく「AIが従うべき命令」として記述する。
具体的には以下を定義する。

- ユースケースの自動判定ルール
- 不足情報があるときの推定・確認のルール
- 登録済みファイルの参照ルール

ファイル構成
-------------------------

```
カスタム指示.md              ← カスタム指示欄に貼り付ける。動作命令を記述する
README.md                    ← ユースケース棚卸し・セットアップ手順・日常利用例
files/メール返信.md          ← 構成ルール・トーン判定・出力例
files/チャット返信.md        ← 簡潔さの基準・絵文字・改行ルール・出力例
files/SNS投稿.md             ← 文字数・フック・ハッシュタグ・出力例
files/PR文.md                ← 構成(課題→解決→ベネフィット)・禁止表現・出力例
files/ブログ記事.md          ← 見出し構成・導入パターン・SEO観点・出力例
files/要約・変換.md          ← 要約・言い換え・トーン変換・フォーマット変換の各ルール
```

ユースケースが増えた場合は files/ 以下にファイルを追加し、
README のユースケース棚卸し表を更新する。

各ファイルに必須の記載事項
-------------------------

money-management の実装を参照し、以下を必ず含める。

1. このファイルを参照するタイミング(AIへの指示)
2. 出力フォーマット定義
3. 判定ルール(トーン・文字数・構成などをAIが推定する基準)
4. 出力例(良い例・避けるべき例)
5. 出力手順(AIが従うステップ)

「ルール」と「禁止事項」の箇条書きだけでは情報源として機能しない。
出力例と判定基準がないと、AIは一般論的な出力に戻る。

プロンプトテンプレートの扱い
-------------------------

`プロンプトテンプレート.md` は情報源ファイルとして登録しない。
README に「使い方の例」として記載するにとどめる。
理由: テンプレートは読み物であり、AIへの命令ではないため。

適用範囲と制約
-------------------------

- 対象: 日常的な文章作成(短文〜中程度)
- 対象外: 法律文書・医療文書・高精度な専門文書
- 事実確認はユーザー責任
- 長文の一括生成は非推奨(分割して生成・確認を繰り返す)
