アイデア出し
=========================

このファイルを参照するタイミング
-------------------------

以下のいずれかに該当する場合にこのファイルを参照すること。

- アイデアをたくさん出したい・発想を広げたい
- 既存のものを改善・発展させたい
- 目標や課題を中心に関連アイデアを整理したい
- 「何かいい方法はないか」「他にどんな手があるか」という相談

フレームワークの使い分け
-------------------------

| 状況                                           | 使うフレームワーク   |
| ---------------------------------------------- | -------------------- |
| とにかくアイデアを多く出したい                 | ブレインストーミング |
| 目標・テーマを中心にアイデアを体系的に広げたい | マンダラート         |
| 既存のものを改善・変形・発展させたい           | SCAMPER              |

ブレインストーミング
-------------------------

### 用途

量を重視してアイデアを発散させる。
実現可能性は問わず、多様な視点からアイデアを出す。

### 処理の手順

1. テーマ・目的を確認する
2. 批判なしにアイデアを列挙する(最低10個、可能なら20個以上)
3. 出たアイデアをグループ化・整理する
4. 特に可能性がありそうなものを3〜5個ピックアップする

### 出力フォーマット

```
## アイデア一覧
1. (アイデア)
2. (アイデア)
...

## グループ整理
- (グループ名): (該当するアイデアの番号)
- (グループ名): (該当するアイデアの番号)

## 特に可能性がありそうなもの
1. (アイデア): (理由)
2. (アイデア): (理由)
3. (アイデア): (理由)
```

マンダラート
-------------------------

### 用途

中心のテーマ・目標から関連アイデアを8方向に広げて体系的に整理する。
目標達成に必要な要素や、課題解決のアプローチを網羅的に洗い出すときに使う。

### 処理の手順

1. 中心テーマを確認する
2. 中心テーマに関連する要素を8つ列挙する
3. 各要素についてさらに具体的なアイデア・アクションを展開する

### 出力フォーマット

```
## 中心テーマ: (テーマ)

## 8つの要素
1. (要素): (具体的なアイデア・アクション)
2. (要素): (具体的なアイデア・アクション)
3. (要素): (具体的なアイデア・アクション)
4. (要素): (具体的なアイデア・アクション)
5. (要素): (具体的なアイデア・アクション)
6. (要素): (具体的なアイデア・アクション)
7. (要素): (具体的なアイデア・アクション)
8. (要素): (具体的なアイデア・アクション)

## 最初に取り組むべきこと
(優先度が高い要素・アクションを理由とともに2〜3点)
```

SCAMPER
-------------------------

### 用途

既存の製品・サービス・アイデア・やり方を7つの視点で変形・発展させる。
「今あるものをどう改善できるか」を考えるときに使う。

### 7つの視点

| 視点                    | 問い                                 |
| ----------------------- | ------------------------------------ |
| Substitute(代替)        | 何かを別のものに置き換えられないか   |
| Combine(結合)           | 何かと組み合わせられないか           |
| Adapt(応用)             | 他の分野から応用できるものはないか   |
| Modify(修正)            | 形・大きさ・色・機能を変えられないか |
| Put to other uses(転用) | 別の使い方・別の場面で使えないか     |
| Eliminate(削除)         | 不要な要素を取り除けないか           |
| Reverse(逆転)           | 順番・立場・発想を逆にできないか     |

### 出力フォーマット

```
## 対象: (改善・発展させたいもの)

## SCAMPERによるアイデア

- 代替: (アイデア)
- 結合: (アイデア)
- 応用: (アイデア)
- 修正: (アイデア)
- 転用: (アイデア)
- 削除: (アイデア)
- 逆転: (アイデア)

## 特に可能性がありそうなもの
1. (アイデア): (理由)
2. (アイデア): (理由)
3. (アイデア): (理由)
```

出力の原則
-------------------------

- フレームワーク名はユーザーに対して前面に出さない
- アイデアの良し悪しを断定しない(実現可能性の評価はユーザーに委ねる)
- 情報が不足している場合は「テーマをもう少し教えてください」と1点だけ確認する
