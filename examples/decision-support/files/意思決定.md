意思決定
=========================

このファイルを参照するタイミング
-------------------------

以下のいずれかに該当する場合にこのファイルを参照すること。

- 複数の選択肢があり、どれを選ぶか迷っている
- やるべきか・やめるべきかの判断を求められている
- 優先順位をつけたい
- 個人事業・小規模事業の状況整理や方向性の検討

フレームワークの使い分け
-------------------------

| 状況                                         | 使うフレームワーク |
| -------------------------------------------- | ------------------ |
| 2つ以上の選択肢を比較したい                  | プロコン           |
| やることが多すぎて優先順位をつけたい         | 優先度マトリクス   |
| 判断に不確実な要素(条件分岐)がある           | ディシジョンツリー |
| 個人・事業の強み・弱みを整理したい           | SWOT分析           |
| 市場・競合・自分(自社)の立ち位置を整理したい | 3C分析             |

複数のフレームワークが有効な場合は、最も状況に合ったものを選ぶか、
段階的に組み合わせる(例: SWOT分析で状況を整理してからプロコンで選択肢を比較)。

プロコン
-------------------------

### 用途

2つ以上の選択肢のメリット・デメリットを整理して比較する。

### 出力フォーマット

```
## 選択肢の比較

### 選択肢A: (名称)
- メリット: (箇条書き)
- デメリット: (箇条書き)

### 選択肢B: (名称)
- メリット: (箇条書き)
- デメリット: (箇条書き)

## 整理のポイント
(比較して見えてきた重要な判断軸を2〜3点)

## 判断の参考
(どちらが向いているか、どんな価値観・優先事項を持つ人に向いているかを提示する。
最終判断はユーザーに委ねる)
```

優先度マトリクス
-------------------------

### 用途

複数のタスク・選択肢を「重要度」と「緊急度」(または「効果」と「実現性」)の
2軸で整理して優先順位をつける。

### 出力フォーマット

```
## 優先度マトリクス

### 今すぐやる(重要度: 高 / 緊急度: 高)
- (項目): (理由)

### 計画的にやる(重要度: 高 / 緊急度: 低)
- (項目): (理由)

### 委譲・効率化を検討(重要度: 低 / 緊急度: 高)
- (項目): (理由)

### 後回しでよい(重要度: 低 / 緊急度: 低)
- (項目): (理由)

## 最初に取り組むべきこと
(1位〜3位を理由とともに提示)
```

ディシジョンツリー
-------------------------

### 用途

「もし〜なら」という条件分岐がある判断を整理する。
結果が不確実な場合や、段階的に判断が必要な場合に使う。

### 出力フォーマット

```
## 判断ツリー

(条件と選択肢をインデントで表現する)
判断: (最初の問い)
├ YES -> (次の判断または結論)
│   ├ YES -> (結論)
│   └ NO  -> (結論)
└ NO  -> (次の判断または結論)
    ├ YES -> (結論)
    └ NO  -> (結論)

## 各結論の説明
(それぞれの結論が意味することと、その場合に取るべきアクション)
```

SWOT分析
-------------------------

### 用途

個人・事業の強み・弱み・機会・脅威を整理して、
現状把握や今後の方向性の検討に使う。

### 出力フォーマット

```
## SWOT分析

### 強み(Strengths): 自分(自社)の内部的な優位点
- (箇条書き)

### 弱み(Weaknesses): 自分(自社)の内部的な課題
- (箇条書き)

### 機会(Opportunities): 外部環境の追い風
- (箇条書き)

### 脅威(Threats): 外部環境のリスク
- (箇条書き)

## 整理のポイント
(強みを活かせる機会、弱みが脅威になるリスクなど、特に注目すべき点を2〜3点)
```

3C分析
-------------------------

### 用途

市場(Customer)・競合(Competitor)・自分/自社(Company)の3つの視点から
事業環境を整理して、方向性や差別化を検討する。

### 出力フォーマット

```
## 3C分析

### 顧客・市場(Customer)
- (ターゲットとなる顧客層・ニーズ・市場の特徴)

### 競合(Competitor)
- (主な競合・代替手段とその強み・弱み)

### 自分/自社(Company)
- (強み・リソース・差別化できる点)

## 整理のポイント
(3つの視点から見えてきた機会・課題を2〜3点)
```

出力の原則
-------------------------

- フレームワーク名はユーザーに対して前面に出さない
  (例: 「SWOT分析で整理します」ではなく「強み・弱みと外部環境の視点で整理します」)
- 分析結果は参考情報であり、最終判断はユーザーに委ねる旨を出力末尾に添える
- 情報が不足している場合は推定して処理し、「〜と解釈して整理しました」と補足する
