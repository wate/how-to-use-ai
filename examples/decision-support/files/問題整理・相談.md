問題整理・相談
=========================

このファイルを参照するタイミング
-------------------------

以下のいずれかに該当する場合にこのファイルを参照すること。

- 何が問題かよくわからない・モヤモヤしている
- 問題の原因を掘り下げたい
- キャリア・転職・働き方の相談
- 人生の選択や将来の方向性の相談
- 目標を具体化・整理したい
- 頭の中を整理したい

フレームワークの使い分け
-------------------------

| 状況                                       | 使うフレームワーク |
| ------------------------------------------ | ------------------ |
| 問題の根本原因を掘り下げたい               | なぜなぜ分析       |
| 頭の中のモヤモヤ・断片的な情報を整理したい | KJ法               |
| 目標を具体化・明確にしたい                 | SMARTゴール        |
| キャリア・人生の方向性を整理したい         | Will-Can-Must      |
| 目標に向けた行動計画を立てたい             | GROWモデル         |

### 複数のフレームワークを組み合わせる例

- KJ法でモヤモヤを整理 -> なぜなぜ分析で根本原因を特定
- Will-Can-Mustで方向性を整理 -> SMARTゴールで目標を具体化 -> GROWモデルで行動計画を立てる

なぜなぜ分析
-------------------------

### 用途

「なぜ?」を繰り返すことで問題の表面的な原因から根本原因へと掘り下げる。
「何が問題かはわかるが、なぜそうなるかがわからない」場面で使う。

### 処理の手順

1. 問題(事象)を1文で明確にする
2. 「なぜそれが起きているか」を問い、原因を列挙する
3. 各原因に対して再び「なぜ?」を問う
4. 5回程度繰り返して根本原因に近づく
5. 根本原因に対するアクションを提示する

### 出力フォーマット

```
## 問題: (事象を1文で)

## 原因の掘り下げ
なぜ1: (原因)
なぜ2: (なぜ1の原因)
なぜ3: (なぜ2の原因)
なぜ4: (なぜ3の原因)
なぜ5: (なぜ4の原因)

## 根本原因
(最終的に特定された根本原因)

## 対応のヒント
(根本原因に対して取れるアクションを2〜3点)
```

KJ法
-------------------------

### 用途

頭の中にある断片的な情報・モヤモヤ・不満・アイデアを書き出して
グループ化し、構造を見つけて整理する。
「何から手をつければいいかわからない」場面で使う。

### 処理の手順

1. ユーザーから出てきた情報・キーワードを全て列挙する
2. 似ているもの・関連するものをグループ化する
3. 各グループにラベルをつける
4. グループ間の関係性を整理する
5. 特に重要・緊急なグループを特定する

### 出力フォーマット

```
## 出てきた情報の整理

### (グループ名)
- (情報・キーワード)
- (情報・キーワード)

### (グループ名)
- (情報・キーワード)
- (情報・キーワード)

## グループ間の関係
(グループ同士のつながりや影響関係を説明)

## 特に注目すべきこと
(最も重要・緊急なグループと、その理由)
```

SMARTゴール
-------------------------

### 用途

曖昧な目標を「具体的・測定可能・達成可能・関連性がある・期限付き」の
5つの基準で明確にする。

### 5つの基準

| 基準                 | 内容                                               |
| -------------------- | -------------------------------------------------- |
| Specific(具体的)     | 何を・誰が・どのように、が明確か                   |
| Measurable(測定可能) | 達成したかどうかを数値や事実で確認できるか         |
| Achievable(達成可能) | 現実的に達成できる範囲か                           |
| Relevant(関連性)     | 自分のやりたいこと・求められることと合致しているか |
| Time-bound(期限付き) | いつまでに達成するかが決まっているか               |

### 出力フォーマット

```
## 元の目標
(ユーザーが述べた目標)

## SMART基準での整理
- 具体的: (現状の評価と改善案)
- 測定可能: (現状の評価と改善案)
- 達成可能: (現状の評価と改善案)
- 関連性: (現状の評価と改善案)
- 期限: (現状の評価と改善案)

## 整理後の目標
(SMART基準を満たした形に言い換えた目標)
```

Will-Can-Must
-------------------------

### 用途

「やりたいこと(Will)」「できること(Can)」「求められること(Must)」の
3つの視点でキャリア・人生の方向性を整理する。
転職・キャリアチェンジ・将来の方向性を考える場面で特に有効。

### 出力フォーマット

```
## Will-Can-Must の整理

### Will(やりたいこと)
- (ユーザーが述べたやりたいこと・なりたい姿)

### Can(できること)
- (現在のスキル・強み・経験)

### Must(求められること)
- (仕事・家族・社会から期待されていること)

## 3つの重なりから見えること
(Will・Can・Mustが重なる部分と、ギャップが大きい部分を整理)

## 方向性のヒント
(整理から見えてきた選択肢や次のステップを2〜3点。最終判断はユーザーに委ねる)
```

GROWモデル
-------------------------

### 用途

Goal(目標)・Reality(現状)・Options(選択肢)・Will(行動意志)の
4つのステップで、目標達成に向けた行動計画を整理する。
「何をすればいいかはわかっているが、どう進めるかが決まらない」場面で使う。

### 処理の手順

情報が十分ある場合は一括で整理する。
情報が不足している場合はステップごとに1つずつ確認しながら進める。

### 出力フォーマット

```
## GROWで整理した行動計画

### Goal(目標): 何を達成したいか
(具体的な目標)

### Reality(現状): 今どこにいるか
(現状・課題・使えるリソース)

### Options(選択肢): どんな手があるか
- (選択肢1)
- (選択肢2)
- (選択肢3)

### Will(行動意志): 何から始めるか
- 最初の一歩: (具体的なアクション)
- 期限: (いつまでに)
```

専門家誘導の判定基準
-------------------------

以下に該当する場合は出力末尾に専門家誘導を添えること。

- メンタルヘルス・精神的なつらさに関わる相談
- 健康・身体的な症状に関わる相談
- 法的なトラブル・権利に関わる相談
- 税務・会計に関わる判断
- 労働問題・雇用に関わる相談

出力の原則
-------------------------

- フレームワーク名はユーザーに対して前面に出さない
- 相談内容を否定・批判しない
- 感情的な側面にも配慮しつつ、整理・構造化に集中する
- 最終的な判断・選択はユーザーに委ねる
- 専門家が必要な領域は情報提供にとどめ、専門家への相談を促す
