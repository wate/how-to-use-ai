記録・ふりかえり
=========================

日々の活動・思考を音声や短文で記録し、日次まとめ・週報・月報へと自動的に整理する
活用事例です。

メモを投げるだけで、AIが行動・TODO・約束・保留を仕分けして出力します。

対象プラットフォーム
-------------------------

| プラットフォーム | 想定プラン     | モデル            |
| ---------------- | -------------- | ----------------- |
| ChatGPT          | 無料プラン以上 | GPT-5.5 Instant   |
| Claude           | 無料プラン以上 | Claude Sonnet 4.6 |

ファイル構成
-------------------------

| ファイル                               | 役割                                   |
| -------------------------------------- | -------------------------------------- |
| [カスタム指示.md](カスタム指示.md)     | カスタム指示欄に貼り付ける内容         |
| [files/まとめ.md](files/まとめ.md)     | 日次まとめのフォーマット・仕分けルール |
| [files/レポート.md](files/レポート.md) | 週報・月報の構成・抽象化ルール         |

セットアップ(初回のみ)
-------------------------

> プロジェクト の概要や共通の考え方は [活用事例 README](../README.md) を参照してください。

### 基本的な流れ

1. [プロジェクトを作成](../README.md#プロジェクト機能とは)する(例: 「記録・ふりかえり」)
2. `カスタム指示.md` の内容を指示欄に貼り付ける
3. 登録対象ファイルを各プラットフォームの指定の項目に登録する

- ChatGPT: 「情報源」項目
- Claude: PC版は「ファイル」、スマートフォン版は「プロジェクトナレッジ」

### 登録対象ファイル

ここでの登録対象ファイルはfiles/以下です。

- [files/まとめ.md](files/まとめ.md)
- [files/レポート.md](files/レポート.md)

プロフィールを作成した場合は、必要に応じて追加で登録してください。
プライベートか仕事かの文脈判断が正確になります。

- 推奨: プロジェクトにファイルとして登録する(プロジェクト単位で文脈を変えたい場合)
- フォールバック: アカウントのプロファイル設定に登録する(全チャットで自動適用、プロジェクトにファイルがない場合に使用される)

作り方は [プロフィール生成プロンプト](../../prompts/プロフィール生成.md) を参照してください。

日常利用
-------------------------

### 基本の流れ

```
1. プロジェクト / Gem を選択してチャットを開始する
2. その日の活動・気づき・タスクを思いついたタイミングで投げる
3. 1日の終わりに「今日のログをまとめて」と依頼する
4. 生成された日次まとめをMarkdownとして保存する
```

### ログの投げ方

入力は雑で良い。整理はAIに任せる。

```
午前中に〇〇の対応をした。

△△さんから連絡があって、来週までに見積もりを送ることになった。

冷蔵庫の醤油が少なくなってたので次の買い物のときに買い足す。

〇〇の件、少し気になっているので後で調べる。
```

### まとめの依頼

```
# 日次まとめを作る
今日のログをまとめて

# 週報を作る
今週をまとめて

# 月報を作る
今月をまとめて
```

### 日次まとめのイメージ

```
# 2026-05-11 まとめ

## 行動サマリー
- 〇〇の対応

## TODO・次アクション
- 〇〇の件を調べる
- 醤油を買い足す

## 約束・確認事項
- △△さんへ来週までに見積もりを送る
```

その他の活用事例は[活用事例](#活用事例)を参照。

チャット管理
-------------------------

- 1日1チャットを基本とする
- 日次まとめを生成してMarkdownに保存したらチャットを閉じる
- 週報・月報は日次まとめをまとめて新しいチャットに貼り付けて依頼する

Tips
-------------------------

### 音声入力と組み合わせる

スマホの音声入力で話しかけるだけでログが残せる。
移動中・作業の合間など、手が離せない場面でも継続しやすい。

### 完璧を目指さない

短くても継続することを優先する。
「今日は〇〇した」の1行だけでも十分ログになる。

### 週報・月報の依頼方法

日次まとめのMarkdownをまとめて新しいチャットに貼り付け、以下のように依頼する。

```
以下は今週の日次まとめです。週報にまとめてください。

(日次まとめを貼り付け)
```

活用事例
-------------------------

この仕組みは「ログを雑に投げてまとめる」という構造なので、様々な用途に応用できる。

- 作業ログ: その日の作業内容・進捗・気づきを記録して日次まとめにする
- 読書ログ: 章ごとの気づき・疑問・引用をメモして読了後にまとめる
- 学習ログ: 勉強・講座・動画視聴の内容を記録して定期的にふりかえる
- 読書+学習の統合ログ: 読書と学習を同じチャットに投げてインプットを一元管理する
- 育児ログ: 子供の成長・出来事・体調を記録して月次でふりかえる
- 買い物ログ: 買ったもの・使い切ったものを記録して買い物リストを整理する
- 失敗ログ: うまくいかなかったことを記録して改善のパターンを把握する
- 逆算ログ: 目標から逆算して今日すべきことを記録・ふりかえる

### 事業者向け活用例

- 現場メモ: 作業中に気づいたことや確認事項を音声入力でメモして、日次まとめを見積もり・報告書のインプットにする
- 訪問記録: 顧客訪問後の会話内容・決定事項・次回アクションをその場でログに残す
- 工程ログ: 作業の進捗・トラブル・対応内容を記録して、週次で振り返り資料を作る
