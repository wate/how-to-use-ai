記録・ふりかえり 設計資料
=========================

目的
-------------------------

日記・行動ログの記録や週次・月次のふりかえりを、
最小限の入力で継続しやすくするAIアシスタントの構成を定義する。

対象ユーザー
-------------------------

- 日記・行動ログを習慣化したい個人
- 週次・月次でふりかえりをしたいビジネスパーソン
- 自己改善・習慣化に取り組んでいる人

設計の核心
-------------------------

- ユーザーがやること: 出来事・気づき・感情をメモとして投げるだけ
- AIがやること: 用途を文脈から判定し、登録済みファイルのルールに従って整形・出力する
- ユーザーが毎回入力する必要があるもの: 原則として素材(メモ・箇条書き)のみ

フォーマットや構成をユーザーが毎回指定する設計は採用しない。
AIが文脈と登録ファイルから判断して出力する。

設計方針
-------------------------

### 用途別ファイル分割

「良い出力」の基準がユースケースによって異なるため用途単位でファイルを分割する。
日記と週次ふりかえりでは、構成・問いかけ・文量の定義がまったく違う。

### 継続しやすさを最優先にする

ふりかえりは継続してこそ価値が出る。
入力ハードルを下げることを最優先とし、
フォーマットの厳密さよりも「投げれば出力される」体験を重視する。

### カスタム指示はAIへの動作命令として機能させる

- ユースケースの自動判定ルール(日記 / 週次 / 月次)
- 不足情報があるときの推定ルール(日付が未記載なら「今日」として処理するなど)
- 登録済みファイルの参照ルール

ファイル構成
-------------------------

```
カスタム指示.md              <- カスタム指示欄に貼り付ける。動作命令を記述する
README.md                    <- ユースケース棚卸し・セットアップ手順・日常利用例
files/日記・行動ログ.md      <- 記録フォーマット・出力例・日付補完ルール
files/週次ふりかえり.md      <- 週次構成・問いかけ項目・出力例
files/月次ふりかえり.md      <- 月次構成・問いかけ項目・出力例
```

各ファイルに必須の記載事項
-------------------------

1. このファイルを参照するタイミング(AIへの指示)
2. 出力フォーマット定義
3. 判定・補完ルール(日付・期間・トーンをAIが推定する基準)
4. 出力例(良い例・避けるべき例)
5. 出力手順(AIが従うステップ)

適用範囲と制約
-------------------------

- 対象: 個人の日常記録・ふりかえり
- 対象外: 業務日報・チーム向け報告書など組織向けの記録
- 事実の正確性はユーザー責任(AIは投げられた素材をもとに整形するのみ)
- 長期間のログをまとめて処理する場合は分割して送ることを推奨

設計判断ログ
-------------------------

| 判断                                       | 理由                                                             |
| ------------------------------------------ | ---------------------------------------------------------------- |
| 日記・週次・月次を別ファイルに分割する     | 構成・問いかけ・文量の基準が用途ごとに異なるため                 |
| フォーマット指定をユーザーに求めない       | 入力ハードルを下げて継続しやすくするため                         |
| 業務日報・チーム向け報告は対象外とする     | 組織ルールへの依存が大きく、汎用的な設計が困難なため             |
