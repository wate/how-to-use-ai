記録・ふりかえり 設計資料
=========================

目的
-------------------------

日々の活動・思考を音声や短文で記録し、日次まとめ・週報・月報へと自動的に整理する
AIアシスタントの構成を定義する。

対象ユーザー
-------------------------

- 日記・行動ログを習慣化したい個人(仕事・プライベート問わず)
- 週次・月次でふりかえりをしたいビジネスパーソン
- 自己改善・習慣化に取り組んでいる人

設計の核心
-------------------------

- ユーザーがやること: その時点の状況・思考・感情を雑にメモするだけ
- AIがやること: 用途を文脈から判定し、登録済みファイルのルールに従って整形・出力する
  具体的には以下を雑ログから自動で抽出・仕分けする。
    - 行動サマリー(今日やったこと)
    - TODO・次アクション(次回までにやること。仕事のタスクも買い物メモも同様)
    - 約束・確認事項(誰かとの取り決め・確認が必要なこと)
    - 保留事項(判断待ち・後日対応)
    - 気づき・学び
- ユーザーが毎回入力する必要があるもの: 原則として素材(メモ・箇条書き)のみ

フォーマットや構成をユーザーが毎回指定する設計は採用しない。
AIが文脈と登録ファイルから判断して出力する。

情報圧縮の構造
-------------------------

雑ログを段階的に圧縮・抽象化していく。

```text
雑ログ(音声・短文メモ)
↓
日次まとめ(行動・TODO・約束・保留を仕分け)
↓
週報
↓
月報
```

下位ほど具体的、上位ほど抽象化する。

設計方針
-------------------------

### 1日1チャットを基本単位とする

1日の間は同じチャットを使い続ける。
その日の活動・思考を、区切りごとに投げ込む。
1日の終わりに日次まとめの生成を依頼し、Markdownとして保存する。

ChatGPT / Gemini は「整理・分析エンジン」として扱い、長期保存は行わない。
永続保存はMarkdownを正本とする。

### 継続しやすさを最優先にする

入力は雑で良い。構造化はAI側に任せる。
フォーマットの厳密さより「投げれば出力される」体験を重視する。

### 入力の性質でファイルを分割する

日次まとめとレポート(週報・月報)では入力の性質が根本的に異なるため、ファイルを分割する。

- 日次まとめ: 断片的な雑ログを構造化・仕分けする処理 → 専用ファイルが必要
- 週報・月報: 整形済みの日次まとめを圧縮・抽象化する処理 → 度合いが違うだけで処理は同じ

### カスタム指示はAIへの動作命令として機能させる

- ユースケースの自動判定ルール(日次まとめ / 週報 / 月報)
- 不足情報があるときの推定ルール(日付が未記載なら「今日」として処理するなど)
- 登録済みファイルの参照ルール

ファイル構成
-------------------------

```
カスタム指示.md          <- カスタム指示欄に貼り付ける。動作命令を記述する
README.md                <- セットアップ手順・日常利用例・Tips
files/まとめ.md          <- 日次まとめのフォーマット・抽出・仕分けルール・出力例
files/レポート.md        <- 週報・月報の構成・抽象化ルール・出力例
```

各ファイルに必須の記載事項
-------------------------

1. このファイルを参照するタイミング(AIへの指示)
2. 出力フォーマット定義
3. 仕分けルール(行動・TODO・約束・保留をAIが判定する基準)
4. 判定・補完ルール(日付・期間をAIが推定する基準)
5. 出力例(良い例・避けるべき例)
6. 出力手順(AIが従うステップ)

適用範囲と制約
-------------------------

- 対象: 個人の日常記録・ふりかえり(仕事・プライベート問わず)
- 対象外: 業務日報・チーム向け報告書など組織向けの記録
- 事実の正確性はユーザー責任(AIは投げられた素材をもとに整形するのみ)
- 長期間のログをまとめて処理する場合は分割して送ることを推奨

設計判断ログ
-------------------------

| 判断                                         | 理由                                                           |
| -------------------------------------------- | -------------------------------------------------------------- |
| 日次まとめとレポートでファイルを分割する     | 入力の性質(雑ログ vs 整形済みまとめ)が根本的に異なるため       |
| 週報・月報を1ファイル(レポート)にまとめる   | 整形済みまとめを圧縮・抽象化する処理として共通化できるため     |
| 日次まとめでTODO・約束事項を独立出力する     | 行動ログに埋もれると後から追いかけにくくなるため               |
| フォーマット指定をユーザーに求めない         | 入力ハードルを下げて継続しやすくするため                       |
| 1日1チャットを基本単位とする                 | コンテキスト肥大化を防ぎ、無料プランでも継続可能にするため     |
| ChatGPT / Geminiを整理エンジンとして扱う     | 長期保存はMarkdownに任せ、役割を明確に分離するため             |
| 業務日報・チーム向け報告は対象外とする       | 組織ルールへの依存が大きく、汎用的な設計が困難なため           |
