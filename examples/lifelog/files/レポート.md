週報・月報ルール
=========================

ユーザーから週報・月報の依頼があった場合は、必ずこのファイルのルールに従って処理すること。
対象: 複数の日次まとめを圧縮・抽象化してレポートを生成する

素材の渡し方
-------------------------

週報・月報の生成には、対象期間の日次まとめをまとめて貼り付けて依頼する。

```
以下は今週の日次まとめです。週報にまとめてください。

# 2026-05-09 まとめ
(内容)

# 2026-05-10 まとめ
(内容)
```

ユースケースの判定
-------------------------

依頼内容から対象期間を判定し、対応するセクションのルールを適用する。

| 手がかり                     | 対象期間           |
| ---------------------------- | ------------------ |
| 「今週」「週報」             | 今週月曜日〜今日   |
| 「今月」「月報」             | 今月1日〜今日      |
| 期間が明示されている         | 指定された期間     |
| 判断できない                 | ユーザーに確認する |

出力フォーマット
-------------------------

### 週報

以下の構成でMarkdown形式で出力する。
該当する内容がないセクションは `(なし)` と記載し、省略しないこと。

```md
# {期間} 週報

## 今週のサマリー
- (今週を一言で表すと何だったか)

## 主な行動・対応
- (今週やったこと・対応したこと)

## 持ち越しTODO
- (来週に持ち越すタスク・確認事項)

## 繰り返し出た問題・気になること
- (複数日にまたがって出てきた課題・パターン)

## 気づき・学び
- (今週の発見・反省・改善のヒント)
```

### 月報

以下の構成でMarkdown形式で出力する。
該当する内容がないセクションは `(なし)` と記載し、省略しないこと。

```md
# {年月} 月報

## 今月のサマリー
- (今月を一言で表すと何だったか)

## 主なテーマ・取り組み
- (今月を通じて取り組んだこと)

## 成果・進捗
- (完了したこと・前進したこと)

## 継続課題
- (解決できなかった問題・来月に持ち越す課題)

## 気づき・学び
- (今月の発見・反省・改善のヒント)
```

期間の補完ルール
-------------------------

- 「今週」の記載のみ → 今週月曜日〜今日までを対象期間として処理する
- 「今月」の記載のみ → 今月1日〜今日までを対象期間として処理する
- 補完した場合は出力後に「※ {期間} として処理しました」と添える

出力形式のルール
-------------------------

- 必ずMarkdown形式で出力する(コードブロックで囲まない)
- 各セクションは箇条書きで記載する
- 日次まとめに記載のない内容を補完・推測して付け足さない
- 日次まとめの内容を単純に羅列するのではなく、傾向・パターンに圧縮して記載する

出力例
-------------------------

### 素材(日次まとめ3日分)

```
# 2026-05-09 まとめ
## 行動サマリー
- 〇〇プロジェクトのミーティングに参加
## TODO・次アクション
- △△さんへ見積もりを送る(来週まで)
## 約束・確認事項
- △△さんへ来週までに見積もりを送る
## 保留事項
- ミーティングで出た課題の解決策(未決定)
## 気づき・学び
- 〇〇の件は想定より複雑

# 2026-05-10 まとめ
## 行動サマリー
- 〇〇の追加調査
- 見積もり作成
## TODO・次アクション
- 見積もりの最終確認
## 約束・確認事項
- (なし)
## 保留事項
- ミーティングの課題、解決策の方向性は見えてきたが未確定
## 気づき・学び
- 調査を進めると想定外の論点が出てきた

# 2026-05-11 まとめ
## 行動サマリー
- 見積もりを△△さんへ送付
- ミーティング課題の解決策を仮決定
## TODO・次アクション
- △△さんからの返答待ち
## 約束・確認事項
- (なし)
## 保留事項
- (なし)
## 気づき・学び
- 早めに動いておくと後の調整が楽になる
```

### 出力

```md
# 2026-05-09〜2026-05-11 週報

## 今週のサマリー
- 〇〇プロジェクトの調査・見積もり対応が中心の1週間

## 主な行動・対応
- 〇〇プロジェクトのミーティング参加
- 追加調査・見積もり作成・送付
- ミーティング課題の解決策を仮決定

## 持ち越しTODO
- △△さんからの返答確認

## 繰り返し出た問題・気になること
- 〇〇の件が想定より複雑で、調査のたびに新たな論点が出ている

## 気づき・学び
- 早めに動くと後の調整コストが下がる
- 複雑な案件は調査フェーズを長めに取る必要がある
```

出力手順
-------------------------

1. 依頼内容から週報・月報のどちらかを判定する
2. 対象期間を確認する(記載がない場合は補完ルールに従う)
3. 素材(日次まとめ)全体を読み、傾向・パターンを把握する
4. 対応するフォーマットに従って圧縮・抽象化して出力する
5. 期間を補完した場合は「※ {期間} として処理しました」と添える
