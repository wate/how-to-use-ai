日次まとめルール
=========================

ユーザーからまとめ・日報の依頼があった場合は、必ずこのファイルのルールに従って処理すること。
対象: 日常のログ・行動メモ・読書ログなど、1日〜数日単位で蓄積したログのまとめ

出力フォーマット
-------------------------

以下の構成でMarkdown形式で出力する。
該当する内容がないセクションは省略すること。

```md
# {日付} まとめ

## 行動サマリー
- (今日やったこと・対応したこと)

## TODO・次アクション
- (次回までにやること)

## 約束・確認事項
- (誰かとの取り決め・確認が必要なこと)

## 保留事項
- (判断待ち・後日対応)

## 気づき・学び
- (気づいたこと・学んだこと・違和感など)
```

仕分けルール
-------------------------

ログの内容を以下の基準で各セクションに仕分けする。

| セクション         | 仕分けの基準                                                   |
| ------------------ | -------------------------------------------------------------- |
| 行動サマリー       | すでに完了した行動・対応(「〜した」「〜を読んだ」)             |
| TODO・次アクション | 自分がこれからやること(「〜する」「〜を買う」「〜を調べる」)   |
| 約束・確認事項     | 誰かとの取り決め・相手への連絡・確認待ち                       |
| 保留事項           | 判断できていない・後日対応と明示されているもの                 |
| 気づき・学び       | 感想・違和感・発見・反省・アイデア                             |

1つのログが複数のセクションにまたがる場合は、それぞれに分割して記載する。
該当する内容がないセクションは省略する。

日付の補完ルール
-------------------------

- 日付の記載がない → 今日の日付として処理する
- 補完した場合は出力後に「※ {日付} として処理しました」と添える

出力形式のルール
-------------------------

- Markdown形式でコードブロック(```md)で囲んで出力する
- 各セクションは箇条書きで記載する
- 1項目1行を基本とする
- ログに記載のない内容を補完・推測して付け足さない

出力例
-------------------------

### 素材(日常ログの場合)

```
午前中に〇〇プロジェクトのミーティングに参加した。
△△さんから来週までに見積もりを送ってほしいと依頼があった。
冷蔵庫の醤油が少なくなってたので次の買い物のときに買い足す。
ミーティングで出た課題の解決策はまだ決まっていない。
〇〇の件は想定より複雑だった。もう少し調査が必要かもしれない。
```

### 出力(日常ログの場合)

```md
# 2026-05-11 まとめ

## 行動サマリー
- 〇〇プロジェクトのミーティングに参加

## TODO・次アクション
- △△さんへ見積もりを送る(来週まで)
- 醤油を買い足す
- 〇〇の件を追加調査する

## 約束・確認事項
- △△さんへ来週までに見積もりを送る

## 保留事項
- ミーティングで出た課題の解決策(未決定)

## 気づき・学び
- 〇〇の件は想定より複雑。調査不足だった可能性がある
```

### 素材(読書ログの場合)

```
「〇〇という本」第1章・第2章を読んだ。
第1章: 習慣は小さな行動の積み重ねで形成されるという話が印象的だった。
第2章: 環境設計が習慣化の鍵という主張。自分の作業環境を見直したい。
第3章まで読み進めたい。
```

### 出力(読書ログの場合)

```md
# 2026-05-11 まとめ

## TODO・次アクション
- 第3章まで読み進める

## 気づき・学び
- 習慣は小さな行動の積み重ねで形成される(第1章)
- 環境設計が習慣化の鍵。作業環境を見直す余地がありそう(第2章)
```

### 素材(読書+学習ログの場合)

```
「〇〇という本」第3章を読んだ。
第3章: 意思決定の質は情報量より判断基準の明確さで決まるという話。
Udemyの〇〇講座を1時間受講した。
変数のスコープについて理解が深まった。
関数の引数の扱いがまだ曖昧。次回も続けて受講したい。
```

### 出力(読書+学習ログの場合)

```md
# 2026-05-11 まとめ

## TODO・次アクション
- 〇〇講座の続きを受講する

## 気づき・学び
- 意思決定の質は情報量より判断基準の明確さで決まる(〇〇第3章)
- 変数のスコープへの理解が深まった(〇〇講座)
- 関数の引数の扱いはまだ曖昧。要復習
```

### 素材(逆算ログの場合)

```
今月末までに〇〇の資格試験を受ける。
今日は過去問を10問解いた。正答率は7割。
苦手な分野は〇〇と△△。
今週中に苦手分野の参考書を1周したい。
来週から模擬試験形式で毎日練習する予定。
```

### 出力(逆算ログの場合)

```md
# 2026-05-11 まとめ

## 行動サマリー
- 過去問10問を解いた(正答率70%)

## TODO・次アクション
- 今週中に苦手分野(〇〇・△△)の参考書を1周する
- 来週から模擬試験形式で毎日練習を開始する

## 気づき・学び
- 苦手分野は〇〇と△△。重点的に対策が必要
```

出力手順
-------------------------

1. チャット内のログ全体を対象にする
2. 日付を確認する(記載がない場合は今日の日付で補完)
3. 仕分けルールに従って各セクションに振り分ける
4. 該当する内容があるセクションのみ出力する
5. Markdown形式でコードブロック(```md)で囲んで出力する
6. 日付を補完した場合は「※ {日付} として処理しました」と添える
